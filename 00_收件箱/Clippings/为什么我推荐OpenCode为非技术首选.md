## 桌面版界面介绍

打开 OpenCode（Mac为例，下载应用程序，双击打开），你会看到初始界面：

如果你想对某个文件内容进行处理，直接点击\*\*打开项目\*\*选择文件进入。不选择，请直接点击\*\*/\*\*进入。进入后的界面如下（小窗口）：

主要分为三个区域：

### 1\. 左侧上栏

- **对话列表：你的问题、指令和智能体的思考、执行过程、回复内容**

### 2\. 左侧下栏

- **输入框** ：你给智能体下达命令、询问问题
- 设置选择：智能体模式（build：全权限，编辑文件、执行命令等；plan：只读模式，进行规划，执行需要你确认）、当前使用的AI模型（默认免费模型，可替换）

### 3\. 右侧栏

- 处理的文件和修改内容

## 免费版够不够用？

**我的回答：够用，但有局限性。**

### 免费版的局限

- ⚠️ 高峰期可能需要排队

**对于初学者来说，免费版完全够你入门和日常使用了。**

等你熟悉了，想用得更爽，再考虑付费或配置自己的 API（后期会讲）。

输入和命令

`虽然桌面版是图形界面，但同样支持一些输入命令，非技术端常用命令有：`

- `/new ：创建一个新会话`
- `/open：搜索并打开一个文件`
- `/model：选择一个模型`
- `/agent：选择一个 Agent 执行`

## 给技术人员：TUI 终端版

如果你是一名开发者，喜欢在终端里操作，推荐使用 **TUI 版** ：

### 安装方式

```
# 通过 npm 安装
npm install -g opencode

# 或者通过 cargo 安装
cargo install opencode

# 启动
opencode
```

### TUI 版特点

- 轻量级，占用资源少
- 适合与终端工作流集成
- 支持脚本化和自动化
- 适合习惯命令行的高效开发者
