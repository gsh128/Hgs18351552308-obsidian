# WorkBuddy 使用指南

> **WorkBuddy** 是腾讯推出的全场景职场 AI 智能体桌面工作台。
> 只需用**一句话描述需求**，WorkBuddy 便能像同事一样自主规划和执行复杂任务，并交付可验收的结果。

---

## 一、核心概念

| 概念 | 说明 |
|------|------|
| **自然语言驱动** | 用一句自然语言下任务，无需复杂操作 |
| **自主规划执行** | 自动拆解任务、规划步骤、完成交付 |
| **多任务并行** | 支持同时发起和管理多个任务 |
| **本地文件操作** | 可读取授权的电脑文件夹，进行批量处理 |
| **结果可验收** | 生成的文档、表格、PPT 等可直接使用 |

---

## 二、环境要求

- **网络**：需要网络连接以调用 AI 模型服务
- **账号**：已注册并登录 WorkBuddy 账号

---

## 三、快速上手

### 3.1 进入 WorkBuddy

打开 WorkBuddy，完成账号登录，进入主界面。

### 3.2 发起第一个任务

| 步骤 | 操作 |
|:----:|:------|
| 1 | 在**新建任务页面**的输入框中，用一句话描述需求 |
| 2 | 选择 AI 模型 |
| 3 | 点击发送按钮或按回车键 |
| 4 | WorkBuddy 自动规划并执行任务 |
| 5 | 可在侧边栏继续新建其他任务，并行推进 |

**示例任务描述：**
- "帮我把这份销售数据生成一个分析报告"
- "整理这个文件夹里的图片，按日期重命名"
- "根据这份需求文档生成一个 PPT"

### 3.3 界面结构

| 区域 | 功能 |
|:----:|:------|
| **侧边栏** | 显示任务列表（按文件夹分组），支持搜索和管理 |
| **对话区** | 核心交互区域，消息列表 + 输入框 |
| **结果区** | 右侧面板，展示产物、文件、变更、预览等视图 |

## 四、工作模式

WorkBuddy 支持多种工作模式以适应不同场景：

### 📋 工作模式（默认）
适合日常办公场景：
- 文档生成（工作报告、技术文档、README 等）
- 数据分析（上传数据文件，自动分析和可视化）
- PPT / 报告生成
- 深度研究（复杂问题调研并输出报告）
- 邮件编辑、周报生成等办公场景
- 批量文件处理（整理、重命名、格式转换）

### 💻 编程模式
适合开发场景：
- 日常编码辅助（代码生成、审查、Bug 修复、重构优化）
- 项目理解（快速了解陌生代码库架构）
- 网站 / 应用开发（从需求到代码的全流程辅助）

## 五、高级功能

### 🧩 专家中心
面向不同角色的**领域专家能力**，覆盖多种职业场景。进入"专家中心"可按分类浏览并咨询。

### 🛠️ 技能市场（Skills）
可扩展的 **Skill 技能生态**，用户可以：
- 安装社区共享的技能
- 创建自己的自定义技能
- 通过技能实现"自我进化"

### 🔗 连接器（Connector）
对接外部平台的连接能力，目前支持：
- 腾讯乐享
- QQ邮箱
- TAPD
- 腾讯文档
- 微云

### ⚡ 自动化（Automation）
支持**定时任务**和**周期性任务**：
- 设置每日自动推送资讯简报
- 定时生成周报 / 月报
- 自动化文件处理流程
- 支持一次性任务和循环任务

### 🦞 Claw 远程控制
"龙虾"远程控制能力，可通过手机端（微信/QQ等）远程操控电脑端 WorkBuddy 执行任务。

### 📁 数据管理
管理 WorkBuddy 中的数据资产，包括文件、任务记录等。

## 七、使用技巧

### 有效的任务描述
- **明确目标**：说清楚要做什么、输出什么
- **补充上下文**：必要时指定文件路径、格式要求等
- **分步提出**：复杂任务可分多次对话逐步完善

### 协作技巧
- 多任务并行：不同任务在侧边栏切换查看
- 结果验收：在结果区仔细检查交付物
- 持续追问：对不满意的结果继续提要求优化
