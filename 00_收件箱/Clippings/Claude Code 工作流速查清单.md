# 📋 Claude Code 非编程人员高效工作流速查清单


## 一、 🚀 日常高频必用指令（90%场景覆盖）

| 指令                | 大白话功能         | 非编程适用场景                            | 操作示例                                                   |
| ----------------- | ------------- | ---------------------------------- | ------------------------------------------------------ |
| **`/clear`**      | 彻底清空对话，开启全新会话 | 切换完全不同的任务（比如从写方案→做表格），避免旧内容干扰AI判断  | 输入 `/clear` 回车，直接说新需求                                  |
| **`/compact`**    | 精简对话，保留核心总结   | 长对话（比如多轮改方案）后，让AI聚焦当前需求，不被历史冗余内容影响 | 直接输入 `/compact`，或加自定义要求：`/compact 只保留方案修改的核心要求，忽略无关讨论` |
| **`/web-access`** | 统一联网操作入口      | 查最新资料、抓取网页内容、获取公开数据、搜索行业信息         | 输入 `/web-access` 后说：`帮我搜索2025年房地产行业最新政策`               |
| **`/resume`**     | 恢复历史对话        | 找回之前没做完的任务（比如昨天写的方案、改的文档）          | 输入 `/resume`，选择对应历史会话继续                                |
| **`/fork`**       | 复制当前对话到新会话    | 同一个需求尝试2种不同方案（比如方案A/方案B），互不干扰      | 输入 `/fork`，新会话里直接说「换一个思路写方案」                           |
| **`/cost`**       | 查看当前会话花费和时长   | 监控使用成本，避免超额                        | 输入 `/cost` 回车，直接看本次对话花了多少钱、用了多久                        |

## 五、 🎯 非编程人员专属工作流模板（直接套用）

### 模板1：方案/报告全流程

1. 用 `/clear` 清空对话，避免干扰

2. 用 `/web-access` 查最新行业政策/数据，获取参考

3. 用 `/add-dir` 导入相关历史文档，让AI了解背景

4. 让AI生成初稿，多轮修改后用 `/compact` 精简对话

5. 用 `/review` 审查方案合理性，用 `/security-review` 检查合规风险

6. 用 `/simplify` 优化文案，最终定稿

7. 用 `/cost` 查看本次花费，用 `/fork` 备份方案版本

### 模板2：批量办公任务

1. 用 `/clear` 开启新会话

2. 用 `/batch` 下达批量需求（比如批量生成模板、批量整理数据）

3. 用 `/update-config` 统一格式/规范，确保批量内容一致性

4. 用 `/review` 抽查批量内容质量

5. 用 `/team-onboarding` 把流程做成团队教程，复用效率

### 模板3：团队协作文档

1. 用 `/init` 搭建项目文档框架

2. 用 `/add-dir` 导入团队现有资料

3. 用 `/update-config` 统一团队文档规范

4. 用 `/team-onboarding` 生成团队使用指南

5. 用 `/review` 审查团队文档，确保统一规范

## 六、 💡 非编程人员使用避坑指南

1. **优先用 ** **`/clear`** ** 切换任务**：不同任务（比如写方案→做表格）一定要清空对话，避免AI混淆需求

2. **长对话必用 ** **`/compact`**：超过5轮修改的对话，一定要用这个指令精简，否则AI会记不住核心需求

3. **联网操作必须用 ** **`/web-access`**：所有需要查资料、抓网页的操作，都要通过这个指令，避免AI私自联网

4. **自定义规则用 ** **`/update-config`**：不要每次都重复说格式要求，用这个指令让AI永久记住你的习惯

5. **批量任务用 ** **`/batch`**：不要手动一个个生成，用并行处理大幅提升效率

6. **合规审查必用 ** **`/security-review`**：所有对外/对内的正式文档，都要做一次安全合规检查，避免踩坑
